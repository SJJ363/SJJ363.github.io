# Insurtech Daily

A static, auto-refreshing insurtech news aggregator hosted on **GitHub Pages**
at `https://sjj363.github.io/` (repo `SJJ363.github.io`, served from the default
branch root — committed files are live, no build server).

## How it fits together

- **Data** (`data/`) is regenerated by GitHub Actions (`.github/workflows/news.yml`)
  every ~6h: `fetch-news.js` → `news.json`; `companies.js` → `companies.json`
  (+ `companies-store.json`); `write-brief.js` enhances the brief.
- **The app** is client-rendered from that JSON: `index.html`+`script.js` (the
  wire), `companies.html`+`companies-index.js` (the index).
- **SEO artifacts** are generated by `scripts/seo.js` (runs last in the workflow):
  static per-company pages, `sitemap.xml`, `robots.txt`, and the structured data
  injected into `index.html` / `companies.html`.

## SEO is a build output, not an afterthought — keep it that way

`scripts/seo.js` is the single source of truth for discoverability. Every page a
crawler can reach must have: a unique `<title>` and meta description, a
`<link rel="canonical">`, OpenGraph + Twitter card tags, a `robots` meta, and
appropriate JSON-LD. The `SITE` config and the `head()` builder in `seo.js`
produce all of that consistently — **route new pages through them.**

### Rules when changing the site

1. **New generated page type?** Build its `<head>` via `head()` in `seo.js` so it
   inherits canonical/OG/Twitter/robots automatically, give it page-specific
   JSON-LD, and add its URL to `buildSitemap()`.
2. **New hand-authored page?** Copy the full head block from `index.html`
   (canonical, OG, Twitter, robots, JSON-LD markers) and adapt the values. Add a
   `<!-- SEO:JSONLD -->…<!-- /SEO:JSONLD -->` marker if it needs injected
   structured data, and wire it up in `seo.js`.
3. **Company pages live at `/company/<slug>/`** as real static HTML — fully
   server-rendered (work without JS) and regenerated each run. Never link
   companies via `company.html?c=…`; use `/company/<slug>/`. `company.html` is a
   legacy redirect shim only.
4. **Links and assets use root-absolute paths** (`/style.css`, `/companies.html`,
   `/company/<slug>/`) so they resolve at any URL depth.
5. **Keep pre-rendered content crawlable without JS.** Client scripts may
   *enhance* (search, live times) but the meaningful content and internal links
   must exist in the served HTML first — see the `SEO:COLIST` block in
   `companies.html`.
6. **After changing data shape or page structure, run `node scripts/seo.js`** and
   confirm it reports the expected page/URL counts. It's idempotent — safe to
   re-run; markers are replaced, not appended.
7. **Canonical origin** is `SITE.origin` in `seo.js` (overridable via the
   `SITE_URL` env var). If a custom domain is ever added, change it there and add
   a `CNAME` file — nothing else needs to move.
8. **OG image** is `assets/og.svg` (1200×630). Swapping in a raster `og.png` is a
   one-line change to `SITE.ogImage`; some scrapers (Facebook/LinkedIn) prefer PNG.

### Local check

```bash
node scripts/seo.js            # regenerate pages, sitemap, robots, structured data
python3 -m http.server 8099    # serve root; visit /company/<slug>/ to spot-check
```
